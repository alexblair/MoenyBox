## 2. 交易管理 / Transactions

### 2.1 创建交易

点击首页快捷按钮（支出/收入/转账）或导航到「交易」页面点击"新建"。

必填字段：
- **类型**: 支出 / 收入 / 转账
- **金额**: 正数
- **账户**: 选择资金进出账户
- **日期时间**: 交易发生时间

可选字段：
- **分类**: 选择合适的分类（支持树形选择）
- **对方账户**: 转账交易时需要选择目标账户
- **备注**: 交易说明
- **附件**: 上传图片凭证

### 2.2 交易列表

交易列表页支持：
- **搜索**: 按备注关键词搜索
- **筛选**: 按类型、分类、账户、日期范围、金额范围筛选
- **排序**: 按日期、金额排序
- **分页**: 每页 20 条

### 2.3 批量操作

选中多条交易后可以：
- **批量删除**: 一次性删除多条交易
- **批量编辑**: 批量修改分类、账户等
- **批量克隆**: 批量复制交易（如定期缴费）

### 2.4 重复检测

系统自动检测可能的重复交易（相同金额、相近日期、相同备注），在交易列表页可以查看和处理重复组。

## 3. 分类管理 / Categories

### 3.1 分类层级

分类支持多层级树形结构。例如：

```
生活
├── 餐饮
│   ├── 早餐
│   ├── 午餐
│   └── 晚餐
├── 交通
│   ├── 公交地铁
│   └── 打车
└── 购物
工作
├── 办公用品
└── 差旅
```

### 3.2 管理操作

- **创建**: 设置名称、颜色、图标、父分类
- **编辑**: 修改任意属性
- **删除**: 删除分类（已关联交易的分类不可删除）
- **排序**: 拖拽调整顺序
- **批量操作**: 批量移动分类到其他父分类

### 3.3 推荐分类模板

初始化时系统会提供默认分类模板，也可以手动配置。

## 4. 账户管理 / Accounts

### 4.1 账户类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **银行账户** | 储蓄卡、存折 | 招商银行储蓄卡 |
| **信用卡** | 先消费后还款 | 中信银行信用卡 |
| **虚拟账户** | 电子钱包 | 微信钱包、支付宝 |

### 4.2 账户操作

- **创建**: 设置名称、类型、初始余额
- **编辑**: 修改账户信息
- **归档**: 关闭的账户可以归档，隐藏在主列表
- **余额查看**: 首页可直接查看所有账户余额

### 4.3 账户分组

将账户归类到不同组中（如个人账户、家庭账户、生意账户）。一个账户可以属于多个组。

## 5. CSV 导入 / CSV Import

### 5.1 导入流程

```
1. 准备 CSV 文件 → 2. 上传 → 3. 字段映射 → 4. 预览校验 → 5. 执行导入
```

### 5.2 支持的字段映射

| 系统字段 | 说明 | 示例 CSV 列名 |
|---------|------|--------------|
| `type` | 交易类型 | 类型、收支、type |
| `amount` | 金额 | 金额、交易金额、amount |
| `dateTime` | 日期时间 | 日期、交易时间、date |
| `note` | 备注 | 备注、说明、note |
| `categoryName` | 分类名称 | 分类、category |

### 5.3 智能校验

导入前系统会自动校验每一行数据：
- 标记错误行（格式问题、数据缺失）
- 检测重复交易
- 应用规则引擎自动修正分类/备注
- 匹配分类和账户名称到系统数据

### 5.4 支持格式

CSV 文件编码建议使用 UTF-8，第一行为列标题。

```
type,amount,date,note,category
EXPENSE,99.00,2026-06-12,午餐,餐饮
EXPENSE,15.00,2026-06-12,地铁,交通
INCOME,15000.00,2026-06-10,工资,薪资
```

## 6. 规则引擎 / Rules

### 6.1 什么是规则

规则引擎可以**自动**为交易设置分类、类型等属性。例如：

> **规则**: 如果备注包含"美团" → 自动设置为"餐饮"分类

### 6.2 规则结构

```
规则组 (条件模式: ALL 或 ANY)
  └── 规则 (条件模式: ALL 或 ANY)
       ├── 条件1: 字段 = 备注, 操作符 = 包含, 值 = "美团"
       ├── 条件2: 字段 = 金额, 操作符 = 小于, 值 = "100"
       └── 动作1: 设置分类 = 餐饮
          动作2: 设置类型 = 支出
```

### 6.3 使用场景

- **CSV 导入时**自动分类银行流水
- **手动记账时**自动填充分类
- **批量修复**已有的未分类交易

### 6.4 条件操作符

| 操作符 | 说明 |
|--------|------|
| `contains` | 包含 |
| `equals` | 等于 |
| `not_equals` | 不等于 |
| `starts_with` | 开头是 |
| `ends_with` | 结尾是 |
| `gt` | 大于 |
| `lt` | 小于 |
| `regex` | 正则匹配 |

### 6.5 触发场景

- `IMPORT`: CSV 导入时自动触发
- `MANUAL`: 手动创建或编辑交易时触发
- 可单独控制规则组在哪些场景生效

## 7. 交易模板 / Templates

### 7.1 模板的作用

对于经常发生的交易（如每日咖啡、每月房租），可以创建模板一键记账。

### 7.2 管理模板

- **创建**: 设置名称、类型、金额、分类、账户
- **层级组织**: 模板支持树形分组（如"日常支出 > 餐饮"）
- **启用/禁用**: 可控制模板是否出现在快速记账面板

### 7.3 使用模板

从首页"模板"按钮或交易列表页模板入口，点击模板即可跳转到预填好的记账页面。

## 12. 离线与 PWA / Offline & PWA

### 12.1 PWA 安装

1. 使用 Chrome/Safari 访问 MoneyBox
2. 地址栏会显示安装提示（或使用"添加到主屏幕"）
3. 安装后可以像原生 App 一样使用

### 12.2 离线能力

- 交易数据缓存在浏览器 IndexedDB
- 离线时可浏览已有数据
- 网络恢复后自动同步

### 12.3 响应式设计

- 桌面端: 全功能体验，多列布局
- 移动端: 触屏优化，底部导航

## 13. 常见问题 / FAQ

### Q: 数据安全吗？

数据存储在您自己的 MySQL 数据库中，附件存储在您的服务器本地。我们不会收集任何数据。

### Q: 支持多用户吗？

当前为单用户模式，但数据库模型和权限系统已为多用户预留，未来版本会支持。

### Q: 如何备份数据？

定期备份 MySQL 数据库即可。参考部署文档中的备份脚本。

### Q: 支持哪些币种？

默认人民币（CNY），可通过修改源代码支持其他币种。

### Q: 导入 CSV 报错怎么办？

检查 CSV 编码是否为 UTF-8，列名是否匹配映射字段。预览功能会显示具体的错误信息。
